syn_encoding attribute

syn_encoding attribute specifies the encoding style for the FSM in the design. This attribute takes effect only when FSM Compiler is enabled. You cannot set this attribute at a global level.

Values:

default: mapper automatically picks an encoding style that results in the best performance for the FSM.

onehot: Implements FSM using one flip-flop per state. Only two bits of the state register change and only one of the state register is hot (decide the state).

gray: Implements the FSM using gray coding. Only one of the state register change at a time and all the state register is hot (decides the state).

sequential: The synthesis tool makes no attempt at limiting the number of bits that can change at a time and all the state register is hot (decides the state).

safe: Implements the state machine in the default encoding and adds reset logic to force the state machine to a known state if it reaches an invalid state.

original: Tool implements state machine in the user specified style and does the reachability analysis
