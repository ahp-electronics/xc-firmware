Placing and Unplacing Macros in Design Mode

In the editing area of the main view, macros can be placed automatically or manually. When you use automatic placement (Autoplace All), the system selects appropriate sites for all remaining unplaced components and macros in the design. When you use manual placement, you select both a macro and a site in which to place the macro’s reference component.

When you manually place a macro, you place only the pre-placed components, those components that were placed in the macro library file. Any unplaced components in the macro's library file will have to be placed in a separate operation.

When you Autoplace All components and macros, the system tries to place all of the macro components, whether they are pre-placed or not.

You can also unplace macros. Unplacing a macro unroutes each of the macro's external pins, and then removes the macro from the editing area and adds the macro to the list of the design's unplaced macros.

AutoPlace

You can place all macros automatically through the Autoplace All command.

To automatically place all macros:
In EPIC, choose Place > Autoplace All. All the unplaced components and macros are automatically placed in vacant sites in the editing area. For each macro, any connections that were routed in the macro's library file will be routed when the macro is placed.
Each macro's external pins are routed after the macro is placed, and any non-prerouted macro nets are automatically routed.
Manual Placement

There are two macro manual placement procedures:

Manually placing an unplaced macro in a vacant site.
Moving a macro to another site after it has been placed.
To manually place an unplaced macro:
1. From the Objects menu, select Unplaced Macros, and then select the macro you want to unplace.
2. Select a vacant site in the editing area in which to place the macro's reference component.
Note
The macro's reference component and the selected site must be the same type of block: I/O component and I/O component or logic component and logic component. Note that the preceding two steps may be performed in reverse order.
3. Choose Place > Manual Place. You can also enter place in the EPIC Command Line dialog box. The macro's reference component is placed in the selected site, and all of the macro's pre-placed components (i.e., components placed in the macro's library file) will be placed in sites relative to the reference component. Macro components that are not pre-placed will have to be placed in a separate operation.
Any connections that were routed in the macro's library file are routed when the macro is placed. The macro's external pins are routed after the macro is placed, and any macro nets that were not pre-routed are automatically routed.
Unplacing

The unplace command removes selected macros from their current sites and puts them into the unplaced macros list. The list of unplaced macros can be viewed in the Objects menu.

Before unplacing a macro, the software unroutes each external pin on the macro. Note that locked macros cannot be unplaced.

To unplace selected macros:
1. From the Object menu, select the placed macros that you want to unplace.
2. Choose Design > Place > Unplace.
