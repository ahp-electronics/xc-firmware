EPIC Device Editor

EPIC (Editor for Programmable Integrated Circuits) is a graphical application for viewing and configuring FPGAs. After you have mapped and/or placed and routed an FPGA design, you can open the design file and view and edit it graphically in EPIC Device Editor. Input files for EPIC include .ncd (native circuit description) and .nmc (physical macro) files. Output files include modified .ncd and .nmc files. EPIC contains many advanced features that enable you to quickly and efficiently make design changes. EPIC allows you to apply engineering change orders (ECOs) to a post-place NCD or create custom circuit macros (see Engineering Change Management). The GUI provides a Manhattan-style physical floorplan view and a schematic-like interface for modifying the programming of most device elements. A simple manual route interface can be used to add or modify physical connections. Not all editing actions will result in a legal database, so it is highly recommended that you seek assistance from Lattice technical support before attempting significant changes to an NCD. A physical DRC check is available that will confirm whether a change is legal. EPIC supports all Lattice FPGAs. It can be used for, but is not limited to, the following tasks:

Visually inspect and modify design connections, PFU logic, I/O types, EBR (Embedded block RAM) contents or PLL (phase-locked loops) parameters.
Automatically document the changes that have been made.
Manually place and route critical components before or after running automatic place-and-route tools on an entire design.
Read, write, and undo certain preferences in the physical preference (.prf) file.
Run timing analysis and DRC (design rule check) on your design.

For a quick overview of the EPIC Device Editor interface, see Main Window. The following steps illustrate EPIC usage in the Lattice Diamond design flow.

1. Run your HDL design through the Translate Design, Map, and/or PAR processes. These processes will generate the required physical design (.ncd) file and associated preference (.prf) file for input into EPIC Device Editor.
2. You can import either a mapped physical design (_map.ncd) file or a placed-and-routed physical design (.ncd) file with the associated preference (.prf) file into EPIC, which means that you can edit your design either before or after placement and routing (PAR). For example, you might choose to manually route a critical component before PAR in EPIC. Or you can manually finish PAR if the routing program was unable to route the design to completion.
3. Read, write, or undo preferences in the preference (.prf) file in EPIC when necessary.
4. Run your edited design and preference files through the Export File process for bitstream or JEDEC, depending on the device family.
5. Use the bitstream or JEDEC data for device programming or for troubleshooting the design.
