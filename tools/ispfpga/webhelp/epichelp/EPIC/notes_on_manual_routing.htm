Notes on Manual Routing

Some notes on manual routing:

Objects selected must be part of the same net. For example, you cannot select a net pin that is part of one net and a long line that is part of another net.
If no connection exists between objects you specified, that connection will not be routed, and an error message will appear in the Tcl console.
When you specify objects to be routed together, you must take into account the direction of current flow through the switches connecting the two objects. You can route from source to load or from load to source, but you cannot mix the two directions with one route command run.
When you select an unused net pin or pin wire for a connection, this net pin is added to the net you are routing.
Manual routing routes only the connections you specify. All other unrouted connections on the applicable net remain unrouted.
Partially routed nets that do not terminate at pins (antennas) will be ripped when the Auto Route All command is used unless you lock them down by locking the net. Unlocked antennas also are removed if you run PAR (the Place and Route program) on the design file.
Any segment added to a net with manual routing is locked or unlocked, depending on the net's lock status.
