Object Definitions

The following table describes the types of objects that appear in the editing area of the main view. Those that are preceded with an icon are objects that you can show or hide on the layout by using use the Layers toolbar or the View > Show Layers menu. The View > Show Layers menu also enables you to control the display of text labels and highlighted objects.

Object
Description
Net
Components are connected by nets. A net (also called a signal or a signal net) is a set of two or more component pins to be electrically connected in the finished design. A net normally consists of a driver pin (output or tristate) and one or more load pins (input), but it may have more than one driver pin in certain cases. A net does not pass through a logic block, except in the case of route-throughs.
Net Pin
When a component has been placed but has not yet been routed, the connection between that component's net pins (those pins on the net) and net pins from other components is logical, not physical. The pins are associated with each other, even though there is no electrical connection between them.
Local Wires
Also referred to as “local lines,” local wires exist within the boundaries of the tiles or logic cells and typically carry signals between resources in the cell. They can be employed to carry signals elsewhere by using routing resources like switchboxes.
Long Wires
Also referred to as “long lines,” these are wires that go beyond the boundaries of each individual cell and enable signals to be carried across the chip.
Pin Wires
Pin wires are short lines used to connect two or more net pins.
Switchboxes
Switchboxes are switching matrix resources located at the junctions of horizontal and vertical local lines
Switches
Switches are contact points employed for signal switching between resources on the chip.
Site
Sites are vacant locations on the chip for the potential placement of components.
Components
Components are logical configurations that will, at some point, go into a site.
Routes
Routes are the physical connections between resources on the chip after the design has been routed.
Ratsnests
Ratsnests represent logical connections between associated net pins before routing has taken place,
Macros
Macros define interconnections between logical elements that are designed to perform a specific logical function. You can show or hide the display of macros by using the View > Show Layers menu.
Text
Text refers to the labels that identify sites and placed components. You can turn the display of these labels on or off by using the View > Show Layers menu.
