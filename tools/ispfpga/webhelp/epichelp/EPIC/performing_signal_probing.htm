Signal Probing

The signal probing feature provides a reliable means of routing internal signals to otherwise unused output pins so that they can be observed with test equipment such as a logic analyzer.

When a signal is “probed”, the designer can trace the signal by connecting a logic analyzer to the FPGA output pin. Note that signal probing is not an internal FPGA logic analyzer, nor will it allow for accurate signal timing analysis.

The probed signal list can be created and edited in the Signal Probing Configuration dialog box. You can also save a probed signal list and then reuse it after the .ncd file is regenerated.

To perform signal probing in EPIC:
1. Choose Design > Signal Probing.
2. In the Signal Probing Configuration Dialog Box, begin pairing desired signals by clicking and dragging them from the signal list on the left to potential “receiver” grid cells (available I/O) on the right.
You can sort columns by Available IO, Signal, or Status by clicking the appropriate column title.
Use the Wildcard filter option if you are searching for a particular signal name. For example, type “d” and press the Enter key to filter the list to signals that begin with the letter “d.” To refresh the list to its original state, enter * in the Wildcard filter text box.
To remove signal probes:
Right-click one or more signals in the Probe Configuration grid and select Remove probe, Remove selected probes, or Remove all probes from the popup menu.
To export a probed signal list for reuse:
1. After creating a probed signal list in the Signal Probing Configuration dialog box, click Export to save the current list as a text file.
2. When you want to reuse the same probed signal list, open the Signal Probing Configuration dialog box, and click Import to import the text file you have saved.
Note
You can also manually edit the probed signal list in the text file before importing it to the Signal Probing Configuration dialog box.
