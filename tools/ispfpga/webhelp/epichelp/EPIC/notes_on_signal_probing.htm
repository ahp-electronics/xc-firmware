Notes on Signal Probing

For multiple signal probing, EPIC begins placing signals in the order received. If there are no available sites because a site is occupied (e.g., dropping three signals into a slot between two existing probes), EPIC will find a next available I/O site and continue as long as there are available sites. If EPIC runs out of available viewable sites in the spreadsheet listing in the dialog box despite there being available sites elsewhere, it will “wrap around” to the top and use the next available I/O, starting from the top. A dialog box will appear notifying you of the wrap. It is also possible to probe a single signal more than once, that is, route it to more than one external pin.
When parameters are changed in the Signal Probing Configuration Dialog Box, they are also changed in the editing area of the main view; however, any actions made in the editing area, such as deleting a signal or occupying an I/O site, are not reflected in the dialog box until you click the Refresh button.
Probes will be stored with the design when the design is saved in EPIC, so that the bitstream data file can be generated. The probe components will be saved with a special attribute SIGNALPROBE. When you finish probing, you can reopen the design and delete any or all of the probes you created.
You must reopen the design to make existing probes recognizable in the grid display in the Signal Probe Configuration dialog box.
