Notes on Automatically Routing an Entire Design

When you automatically route the entire design from EPIC, the following operations are performed:

Three iterations of the router
One cost-based cleanup pass
No delay-based cleanup passes

If you want to specify the number of routing iterations to perform instead of accepting the default value (three iterations), you can modify the number of passes by using the Tcl command prompt. The router continues to perform iterations until one of the following is true:

The design is routed to completion and constraints are met
The router completes the number of iterations you specified
The router determines, when no number is specified, that it cannot route the design to completion or meet constraints.

When you route an entire design within EPIC, no existing routing is ripped up (that is, unrouted) and rerouted to accommodate any new routing. Since the PAR (Place and Route) program rips up routes if necessary, routing done by running PAR will produce different results than routing done in EPIC. PAR routes more efficiently, and a densely packed design can more easily be routed in PAR than in EPIC.
