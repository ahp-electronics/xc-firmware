Notes on Adding Macros

When a macro is added, the names of all components, nets and external pins within the macro will have the format macro_name/object_name (e.g., $adder_3/$comp_0 or $decoder_2/$extpin_0).
All components, nets and macros have their own name space and all components, nets and macros must have unique names within their own name space. For example, you can have a component and a macro both of which are named “FRED,” but you cannot have two macros named “FRED” or two nets named “FRED.” EPIC prevents you from entering non-unique names within a name space. Also, the name must not be empty.
When you first instantiate a macro library file into the design, a copy of the macro library file is added to the design file. Thereafter, all subsequent instantiations of this macro refer to the local copy, not the original library file. If the original (external) macro library file is changed or deleted, this will have no effect on any instances of this library file in your design.
If you change the external macro library file and want to update an existing design to use this revised library file, you can do one of the following things:
Re-map the design
In EPIC, delete all instances of the revised macro library file, and then add new instances of the revised macro library file in place of the old instances.
