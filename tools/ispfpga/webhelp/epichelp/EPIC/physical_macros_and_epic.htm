Physical Macros

You can design physical macros using the EPIC design editor. A physical macro (often referred to as “macro” or “macro library file”) is stored in a file with an .nmc extension. It describes a function in terms of logic and connectivity and how the function is mapped to the components of a particular FPGA device architecture. Physical macros can also contain placement and routing information for the function. A macro can be unplaced, partially placed or fully placed, and it can also be unrouted, partially routed or fully routed. Physical macros are particularly helpful if you know the type of device to which you will map the design even before you design the schematic.

A physical macro is created when EPIC is in macro mode. You can also save a design as a macro. See Adding a Macro in Design Mode.

After you create a macro library file, you can instantiate the macro into a design in EPIC. When you instantiate a macro into a design, you copy the contents of the library file into the design file. After it is in the design file, the macro has lost its link to the library file, and if you change the library file, these changes will not be reflected in the instantiated macro in the design file.

You can also instantiate a macro into a schematic drawing by entering a block in the schematic, configuring the block appropriately, and placing a reference to the .nmc file in the schematic. For more information about adding macros to schematics, see the appropriate CAE vendor documentation or contact customer support. Also check with technical support about cross architecture compatibility.
