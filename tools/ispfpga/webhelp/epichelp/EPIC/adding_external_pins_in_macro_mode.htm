Adding External Pins in Macro Mode

External pins connect the instantiated macro to other components in the design. You define the external pins in the macro library file in macro mode, and the pins cannot be modified in a macro instance.

To add external pins to the macro library file:
1. In the editing area of the main view, select the component pins that you want to define as external pins. Use the Ctrl+click combination to select multiple pins.
2. Choose Design > Macro > Add ExtPin.
The External Pin Attributes Dialog Box opens, displaying the name and location of the pin, as well as a default external pin name.
If you selected multiple pins in Step 1, the dialog box displays the attributes for the pin you selected last.
3. If desired, type a new name in the text box for the external pin, and then select a type from the list. See Pin Types.
4. If you selected only one pin in Step 1, click OK to apply your changes and close the dialog box. If you selected multiple pins in Step 1, click Apply to apply your changes to the one pin and proceed to the next.
5. When you have defined all the external pins, click OK.
The selected pins are now external pins, and their names appear in the Tcl console.

When you add an external pin, the name assigned to the new pin is in the format $extpin_<number>, where <number> is a number assigned to each new external pin. Numbering starts at 0, and increases by one for each new added external pin. For example, if you add multiple external pins the first one is called $extpin_0, the second is called $extpin_1, and so on.

For macro nets connected to external nets, use these rules to declare and name external pins, and to set their TYPE attributes:

If a macro net is an output to the external circuitry (that is, one or more pins on the net are drivers for the external net), and the macro net has both input and output pins, declare one output (driver) pin as external. If there are multiple output pins on the net, it does not matter which one you select.
If a macro net is an input to the external circuitry (that is, one or more pins on the net act as loads for the external net), and the net has both input and output pins, declare one input pin as external. If there are multiple input pins on the net, it does not matter which one you select.
If you are going to use the macro library file in a schematic, you must give the external pins in the macro library file the same names as the pins on the symbol that will instantiate the macro in the schematic.
To ensure that timing analysis works correctly when the macro is instantiated, you might have to set each external pin's TYPE attribute when you add the external pin to the library file.
