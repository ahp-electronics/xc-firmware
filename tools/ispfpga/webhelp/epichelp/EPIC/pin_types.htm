Pin Types

Pin type determines how paths passing through the pin are defined for timing analysis. Pin types are listed in the following table: 

PIN TYPE
Input / Output
Description
INPUT
input pin
specifies that the pin is an input to combinational logic in the macro; it will not be clocked.
CLOCK
input pin
specifies that the pin is used as a clock signal within the macro.  
SRDIRECT
input pin
specifies that the pin is used as an asynchronous set or reset signal within the macro.  
ENABLE
input pin
specifies that the pin is used as an asynchronous control line.
SYNCINPUT
input pin
specifies that the pin is an input that is clocked within the macro (for example, a register input). 
SYNCENABLE
input pin
specifies that the pin is used as a control line synchronous to a clock within the macro.  
SETDIRECT
input pin
specifies that the pin is used as an asynchronous set signal within the macro.
RSTDIRECT
input pin
specifies that the pin is used as an asynchronous reset signal within the macro.
SELECT
input pin
specifies that the pin is used as select signal of multiplexer within the macro.
OUTPUT
output pin
specifies that the pin is an output from combinational logic within the macro; not clocked within the macro. 
SYNCOUTPUT
output pin
specifies that the pin is an output from clocked logic within the macro. The pin is synchronous with respect to the macro clock.
DIRECTIN
output pin
specifies that the pin should be treated as a direct input pin, that is, a pin with no internal delay. This pin type should not be specified for an external macro pin.
