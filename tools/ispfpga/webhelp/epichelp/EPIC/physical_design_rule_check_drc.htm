Physical Design Rule Check (DRC)

Physical Design Rule Check (DRC) is a series of tests to discover logical and physical errors in the design. Physical DRC is applied to EPIC and BITGEN (the bit generation program).

You can run DRC in EPIC by selecting the command from the Actions panel or the Design menu. DRC also runs automatically whenever you make changes in the Logic Block view, whenever you do manual routing, and whenever you add pins to a net. In these instances, the DRC run is transparent.

Results of the DRC are written into the Tcl console. DRC messages can be error messages (for conditions where the routing or component logic does not operate correctly) or warnings (for conditions where the routing or logic are incomplete).
