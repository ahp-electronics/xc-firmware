Notes on Automatically Routing Selected Objects

Some notes about automatically routing selected objects:

When you autoroute a net pin, the amount of routing performed depends on whether the net pin is an output pin (a net driver) or an input pin (a net load). If the pin is a net driver, Auto Route routes the entire net. If the pin is an input pin, Auto Route only routes the connection between the pin and the net driver.
When you autoroute a component, each net pin on the component is routed.
When you autoroute a net or a ratsnest line, the entire net is routed.
When you autoroute selected objects within EPIC, no existing routing will be ripped up (that is, unrouted) and rerouted to accommodate any new routing. Also, no component pins will be swapped for a better routing. Since the PAR (Place and Route) program will rip up routes and swap pins if necessary, routing done in PAR will be more efficient than routing done in EPIC.
An optional use of Auto Route allows you to select two objects on a single net and Auto Route between these objects. These can be any combination of the following objects used by the net: net pin, route segment, local line, long line or pin wire. When you then perform the autoroute command, the software attempts to route a path between the two objects.
