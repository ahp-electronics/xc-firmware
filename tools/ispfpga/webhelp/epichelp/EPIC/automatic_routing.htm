Automatic Routing

The Auto Route option automatically routes any unrouted nets created as a result of an editing action, such as placing a component, creating a new net, or swapping components. Auto Route is performed at the end of each add, copy, swap, and place command. You can also select Auto Route for selected objects.

Auto Route is enabled by default. You can use manual routing to route selected objects by choosing Design > Route > Manual Route. Only Auto Route can be used to route the entire design.

The following objects can be automatically routed:

Nets
Net pins
Ratsnest lines
Components
Macros

Auto Route can also be used between two resources. If both resources are used, they must be on the same net:

Route segments
Used wires
Used or unused component pins
To automatically route selected objects:
1. Select the objects you want to automatically route.
You can select any combination of nets, net pins, ratsnest lines and components and macros. Unrouted nets can only be selected from the Object menu.
2. Choose Design > Route > Auto Route or click the autoroute button on the Actions panel.
The objects you selected are routed (if possible). The display for each routed connection changes from a ratsnest display to a routed net display. If an object could not be routed completely, an error message will appear in the Tcl console and the ratsnest display will remain.
To automatically route the entire design:
Choose Design > Route > Auto Route All.
All unrouted connections are routed (if possible). The display for each routed connection changes from a ratsnest display to a routed net display. If a net cannot be routed to completion, an error message will appear in the Tcl console.
