Running Design Rule Checks (DRC)

You can run DRC for the whole design or for selected objects. A quick way to do this is to select the DRC command from the Actions panel or from the Design > DRC menu. You can also run DRC with specific options by specifying them in the Design Rule Check configuration dialog box.

To run DRC with your current settings:
1. Select the objects on which you will run the DRC.
You can select any combination of components, net pins, route segments, or nets. You can also run DRC with nothing selected, if you want to check all objects in the design.
2. Choose Design > DRC > DRC or click the drc button on the Actions panel.
The DRC results are shown in the Tcl console.
To run DRC with specified options:
1. Select the objects on which you will run the DRC, or select no objects if you want to run DRC on the whole design.
2. Choose Design > DRC > Configure DRC.
3. In the DRC dialog box, select the desired options.
4. Click OK.

The DRC tests are performed on all objects or the objects you selected. The tests follow these rules:

When a component is selected, the nets associated with it will also be checked.
When a pin is selected, the associated component will be checked.
When a combination of nets and components is selected, both net check and block check are performed.

For a list of the individual violations for net and block check, along with the type of message (error or warning) that appears in the Tcl console, see the appropriate documentation on DRC error messages and warnings.
